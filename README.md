# Проект: Руководство по Space Station 13 Colonial Marines (PVE)

Этот файл служит отправной точкой для понимания структуры и целей проекта. Предназначен для разработчиков и AI-ассистентов.

## Описание Проекта
Проект представляет собой руководство для новичков по игре **Space Station 13**, а именно для режима/сервера **Colonial Marines** с упором на **PVE** (Player vs Environment).

**Язык проекта:** Русский

## Основные Принципы
1.  **Целевая Аудитория:** Новички, желающие разобраться в механиках Colonial Marines PVE.
2.  **Язык:** Вся документация и контент ведутся на русском языке.
3.  **Формат:** Статический сайт, генерируемый с помощью MkDocs.

## Технический Стек
*   **Движок:** [MkDocs](https://www.mkdocs.org/)
*   **Тема:** [Material for MkDocs](https://squidfunk.github.io/mkdocs-material/)
*   **Конфигурация:** `mkdocs.yml`

## Структура Проекта
*   `mkdocs.yml` - Основной файл конфигурации MkDocs.
*   `docs/` - Директория с исходными файлами документации в формате Markdown.
    *   `index.md` - Главная страница.
    *   `getting-started/` - Раздел для начинающих.
    *   `lore/` - Лор и описание мира.
    *   `mechanics/` - Описание игровых механик.
    *   `faq.md` - Часто задаваемые вопросы.

## Команды для Разработки
*   `mkdocs serve` - Запуск локального сервера для просмотра изменений в реальном времени.
*   `mkdocs build` - Сборка статического сайта в директорию `site/`.

## Сборка и Просмотр (Building & Previewing)

Для локальной разработки и просмотра изменений перед отправкой (push) используйте следующие команды.

### Быстрый старт (Windows Batch Files)
Для упрощения работы в корне проекта созданы `.bat` файлы:

*   **`run_local.bat`** (Рекомендуется): Сначала собирает проект (Build), затем запускает локальный сервер.  
    *Используйте этот файл для начала работы.*
*   **`build.bat`**: Только собирает статический сайт в папку `site/`.

### Ручные команды (Manual CLI)
Если вы предпочитаете использовать терминал:

1.  **Установка зависимостей** (если требуются):
    ```bash
    pip install mkdocs-material mkdocs-glightbox
    ```

2.  **Запуск локального сервера**:
    ```bash
    mkdocs serve
    ```
    После запуска перейдите по адресу `http://127.0.0.1:8000`. Сервер автоматически обновляет страницу при сохранении файлов.

3.  **Сборка проекта** (Build):
    ```bash
    mkdocs build
    ```
    Эта команда создаст папку `site/` с готовым HTML-кодом сайта.

## Краткий Справочник по Markdown (Markdown Guide)

MkDocs использует Markdown для форматирования текста. Вот основные элементы:

### Заголовки
```markdown
# Заголовок 1
## Заголовок 2
### Заголовок 3
```

### Текст
*   **Жирный**: `**текст**`
*   *Курсив*: `*текст*`
*   `Код`: `` `код` ``

### Списки
Маркированный:
* Элемент 1
* Элемент 2

Нумерованный:
1. Первый
2. Второй

### Ссылки и Картинки
*   Ссылка: `[Текст ссылки](http://example.com)`
*   Внутренняя ссылка: `[На главную](index.md)`
*   Картинка: `![Описание](images/image.png)`

### Блоки предупреждений (Admonitions)
MkDocs Material поддерживает красивые блоки:
```markdown
!!! note "Примечание"
    Текст примечания.

!!! warning "Внимание"
    Важное предупреждение.
```

### Таблицы
```markdown
| Заголовок 1 | Заголовок 2 |
| ----------- | ----------- |
| Ячейка 1    | Ячейка 2    |
```

Более подробную информацию можно найти в [официальной документации MkDocs](https://www.mkdocs.org/user-guide/writing-your-docs/).
